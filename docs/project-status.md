# Project Status

This page tracks status at the module level. It is intentionally brief; detailed design and work notes belong in the module pages.

## NexusUI

Active. The framework has a useful control set and is being hardened around layout, clipping, focus, popups, window management, skins, and retained-control behavior.

## NexusSchema

Active. The project owns schema structure, metadata, command-line behavior, templates, and generation targets.

## NexusLS

Active. The language server is replacing legacy Pascal language-server behavior with Nexus-owned services, tests, caching, diagnostics, and LSP responses.

## NexusTest

Active. The framework supports test suites/cases and a shared-library boundary used by test modules. NexusTestUI provides a visual runner.

## NexusLib

Active support module. Shared runtime code should be documented here when it is used across multiple Nexus projects.

## Documentation

In progress. The immediate documentation goal is to make the site match repository ownership boundaries, remove duplicate doc trees, and replace placeholder pages with repo-backed content.
